Introdução

Dos temas apresentados, escolhemos o Pré-Processador para LaTeX e HTML pois pareceu-nos um bom desafio para aplicar os conhecimentos adquiridos nas aulas de Processamento de Linguagens. Foi assim que nasceu o flexLuthor, o programa capaz de apartir de texto pré formatado convertê-lo para LaTex e/ou HTML.Para tal é necessário criar tags simples e independentes de forma a serem interpretadas e convertidas para vários tipos de documentos finais. Tendo como base um ficheiro com tags conhecidas, o Luthor analisa e converte para o tipo de ficheiro final indicado pelo utilizador. Tudo isto usando a ferramenta FLex para analisar o ficheiro fonte detetando padrões.

Invocação inicial

Só é possível definir o tipo de ficheiro de output através dos argumentos com que se inicia o programa. Existem:
-l -o nome-do-ficheiro - Imprime para o ficheiro indicado em formato LaTeX
-h -o nome-do-ficheiro - Imprime para o ficheiro indicado em formato HTML
-lh -o nome-do-ficheiro - Imprime para 2 ficheiros separados em LaTeX e HTML
-hl -o nome-do-ficheiro - Imprime para 2 ficheiros separados em LaTeX e HTML
--v ou --version - Mostra a versão do programa

Previsões do trabalho futuro

Futuramente poderíamos incluir mais formatos de output como o XML ou RTF e mais tags de formatação de forma bastante simples pois é apenas necessário um ficheiro para cada e acrescentar as respetivas funções de conversão.As tags por nós criadas foram pensadas a serem legíveis para que o utilizador detete logo a sua função, pretendemos depois que estas sejam mais curtas para diminuir a sua presença no texto fonte.

Conclusão

Finalizado o projeto, podemos concluir que ficámos bastante satisfeitos com o resultado final, conseguindo fazer tudo o que foi proposto. Os resultados obtidos através dos testes feitos por nós demonstraram-se dentro do que estaríamos à espera.Este projeto é bastante útil quando é preciso redigir um texto com formatação em mais do que um formato diminuindo o tempo perdido a fazer o mesmo trabalho em cada ficheiro.Este tipo de abordagem permitiu-nos um ponto de vista diferente sobre como abordar padrões em textos, sendo isto bastante enriquecedor para nós. A priori se nos fosse apresentado este enunciado provavelmente faríamos com iterações sucessivas a todas as linhas ficando o código ilegível e o erro era constante. O Flex veio dar uma ajuda impressionante no processamento de linguagens no qual nunca tivémos contato.
